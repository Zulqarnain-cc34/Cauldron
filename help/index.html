Run the environment

git clone <your-repo>
cd project
npm start

Then open http://localhost:3456/. No compile step — ES modules + static server.

  • Left — material brush (searchable)
  • Right — rules toggles (searchable; includes plugins)
  • Space — pause · R — reset world

Built-in plugin: Grenade

Press G or middle-click to throw a grenade toward the cursor. Toggle Grenade blast in Rules.

Documented automatically in Live docs → Grenade with runnable tests.

Layered library

Cauldron is built in layers. Plugins sit on top and import only the public SDK:

import { Species, registerPlugin } from '../js/cauldron/index.js';

See ARCHITECTURE.md for the full layer diagram (L0 Kernel → L4 SDK → L6 Plugins).

Build your own extension

  1. Copy plugins/_template/ to plugins/your-name/
  2. Implement setup() + optional behaviors[] tests
  3. Add one import in plugins/index.js
  4. Reload — your plugin appears in Rules + Docs + npm test

Read plugins/README.md for the full plugin API.

Add a new material (core)

  1. Add species in js/catalog/species.js
  2. Add entry in js/catalog/materials.js
  3. Create js/rules/materials/your.js with behaviors[]
  4. Register in js/sim/manifest.js

Docs and brush picker update automatically from the registry.

Release & tests

Before production, run one command — no need to click through every test:

npm run release

Green READY FOR RELEASE means headless tests, quality gates, catalog coverage, and golden snapshots for all behaviors passed.

While coding: npm run test:watch in a background terminal. See TESTING.md.

Documentation & tests

Every behaviors[] entry is the spec — write it once in your rule or plugin file.

  • npm test — headless (same as CI)
  • npm run test:watch — reruns while you edit (no browser needed)
  • Live docs — searchable catalog + runnable demos
  • Visual runner — optional, for debugging only

See TESTING.md for the integration checklist and inspect() hook.
